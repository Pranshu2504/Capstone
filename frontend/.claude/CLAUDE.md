# frontend/ — ZORA React Native Mobile App

## Purpose
This folder is the React Native mobile app. It calls the backend REST API and renders the full ZORA experience — wardrobe management, virtual try-on, outfit planning, and social feed.

## Actual Stack (from package.json / codebase)
- **Language**: TypeScript 5.9.2
- **Framework**: React 19.1.0, React Native 0.81.5
- **Navigation**: React Navigation v7 (bottom tabs + native stack)
- **Styling**: React Native `StyleSheet` — no external CSS framework. All styles use the token system in `src/constants/colors.ts`
- **Server state**: TanStack Query (React Query) v5 — all API calls go through `useQuery` / `useMutation`
- **Local state**: React Context API + `AsyncStorage` (theme persistence, auth token storage)
- **Animations**: React Native Reanimated v4 — used for complex UI (orbital nav, modals, toggles, door animation)
- **Icons**: `react-native-vector-icons` (Feather set)
- **Input**: `react-native-image-picker`, `react-native-keyboard-controller`
- **Haptics**: `react-native-haptic-feedback`
- **Gestures**: `react-native-gesture-handler`

## Design System
- **App name displayed to users**: ZORA
- **Dark theme**: brass/gold accent `#C9A84C` on charcoal `#090909`
- **Light theme**: sage green `#7A8B6F` on warm beige `#F2EDE4`
- **Fonts**: Playfair Display 700 (headings), Inter 400/500/600/700 (body), Cormorant Garamond 400/600/700 (display)
- All color tokens live in `src/constants/colors.ts` — never hardcode hex values in component files
- Use `useColors()` hook to access the active theme's palette
- Use `<ZoraText>` component for all text (7 variants: display, heading, subheading, body, caption, brass, label)
- Use `<BrassButton>` for all buttons (variants: solid, ghost, outline; sizes: sm, md, lg)

## Key Rules
1. **No hardcoded API URLs** — use `API_BASE_URL` from environment/config, never a raw `http://...` string.
2. **No inline styles** — use `StyleSheet.create()`, reference `useColors()` tokens, never `style={{ color: '#fff' }}`.
3. **TanStack Query for all API calls** — no raw `fetch` or `axios` in component bodies; wrap in `useQuery`/`useMutation`.
4. **No `console.log` in production code** — use `__DEV__` guard if you must debug.
5. **Clean up Three.js / animation resources in `useEffect` cleanup** — prevent memory leaks.
6. **Platform-aware padding** — use `Platform.OS` for iOS/Android differences; check `StatusBar.currentHeight` for Android.
7. **Haptic feedback on all primary button taps** — `HapticFeedback.trigger('impactLight')`.

## Mock Data → Real API Migration
All screens currently use mock data from `src/constants/mockData.ts`. When connecting to the real backend:
- Replace direct imports of `mockData` with TanStack Query hooks
- Auth token stored in `AsyncStorage` under key `@zora_auth_token`
- Query client is initialized in `App.tsx` — add `defaultOptions` for retry/stale time there

## 3D Avatar (Planned Feature)
The virtual try-on 3D avatar viewer is not yet implemented in the frontend. When building it:
- Use `@react-three/fiber` + `@react-three/drei` (React Native compatible via `expo-gl` or `react-native-wgpu`)
- Or use React Native's `WebView` pointing to a Three.js canvas served locally
- GLB meshes are generated by the `ai/` service and served via the backend
- See `frontend/.claude/skills/frontend-3d/SKILL.md` for implementation guide
